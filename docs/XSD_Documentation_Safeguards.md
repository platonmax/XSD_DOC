# Инструкция: как не терять описания (xs:documentation) при правках XSD

- Не менять содержимое `xs:annotation/xs:documentation` и `xs:appinfo` без прямого запроса. Любой текст внутри - только по явному указанию.
- При переносе или переименовании элементов/типов переносить их `xs:annotation` целиком вместе с узлом.
- Замены выполнять только по атрибутам (`name`, `type`, `base`, `ref`, `xpath`), не трогая текст узлов. Не использовать грубый поиск/замену по строкам, если она может задеть документацию.
- При массовых переименованиях использовать скрипт, который меняет атрибуты и пути, но не редактирует тело `xs:documentation`.
- Сохранять кодировку/формат исходника (UTF-8), не запускать автоформатирование, если оно может убирать пробелы или переносы в документации.
- После правок проверять: валидность XML/XSD, diff по количеству/содержимому блоков `xs:documentation` и отсутствие удалений/обрезаний текста.
- Если документацию трогали по задаче - фиксировать это в отчёте отдельно.
