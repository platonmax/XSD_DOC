## Синхронизация с D:\GIT_GGE\xsd (схемы/визуалки)

Короткий сценарий для команды «Сделай синк в ГГЭ».

1. Проверка состояния:
   - `cd D:\GIT_GGE\xsd`
   - `git status` должен быть чистым; если есть чужие изменения — остановиться и спросить.
2. Копирование (ТОЛЬКО схемы/визуалки):
   - Источник: `D:\MAX\projects\XSD_DOC\` (подкаталоги схем).
   - Приёмник: зеркальный путь в `D:\GIT_GGE\xsd\`.
   - Копируем ТОЛЬКО XSD/XSL (и в рабочих каталогах, и при обновлении архива), любые HTML/PDF/DOCX/примеры XML/снапшоты/вспомогательные файлы НЕ копируем.
   - Перед копированием убедиться, что исходный файл новее/нужнее (версия выше или дата свежее). Если в целевой репе файл новее/сомнительно - остановиться и уточнить, ничего не перезатирать.
   - Архивные версии ГГЭ лежат в `D:\GIT_GGE\xsd\Archive\<Схема>\` (плоско по файлам: XSD/XSL с версией в имени). Архив обновляем только если требуется зафиксировать архивную версию схемы или визуалки (XSD/XSL).
3. Контроль после копирования:
   - `git status` должен показать изменения только по ожидаемым схемам/визуалкам.
   - Проверить, что никакие файлы не удалены/переименованы лишним образом.
4. Коммит:
   - Формат сообщения: `<TYPE>(<SCOPE>): <кратко что сделано>`.
   - `TYPE` - по общепринятым префиксам (conventional commits): `feat`, `fix`, `chore`, `refactor`, `docs`, `test`, `build` и т.п.
   - `SCOPE` - схема/область, например `ComparativeQuantityTakeoff` или другая копируемая схема.
   - Пример: `feat(ComparativeQuantityTakeoff): обязательный fileCheckSum (CRC32)`.
5. Pull/Push:
   - `git pull`; при конфликтах - остановиться и сообщить.
   - `git push` после успешного pull и отсутствия конфликтов.
6. Контроль ошибок:
   - Любые ошибки/предупреждения при `git commit`/`git push` фиксировать и обязательно сообщать (включая текст ошибки). Не игнорировать сообщения об отказах/конфликтах.

Если на любом шаге возникает неопределённость (новее/старее, конфликт, лишние изменения) — остановиться и запросить решение.
