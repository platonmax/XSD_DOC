## Синхронизация с D:\GIT_GGE\xsd (ComparativeQuantityTakeoff)

Короткий сценарий для команды «Сделай синк в ГГЭ».

1. Проверка состояния:
   - `cd D:\GIT_GGE\xsd`
   - `git status` должен быть чистым; если есть чужие изменения — остановиться и спросить.
2. Копирование (ТОЛЬКО схемы/визуалки):
   - Источник: `D:\MAX\projects\XSD_DOC\ComparativeQuantityTakeoff\MINSTROY_VERSION\`
   - Приёмник: `D:\GIT_GGE\xsd\ComparativeQuantityTakeoff\`
   - Разрешено копировать только XSD/XSL/HTML схем и визуалок (например: `ComparativeQuantityTakeoff.xsd`, `types/BaseTypes.xsd`, `types/CommonTypes.xsd`, `ComparativeQuantityTakeoff.xsl`, `*.html` визуализации). Примеры XML, снапшоты, вспомогательные файлы НЕ копируем.
   - Перед копированием убедиться, что исходный файл новее/нужнее (версия выше или дата свежее). Если в целевой репе файл новее/сомнительно — остановиться и уточнить, ничего не перезатирать.
3. Контроль после копирования:
   - `git status` должен показать изменения только по ожидаемым схемам/визуалкам.
   - Проверить, что никакие файлы не удалены/переименованы лишним образом.
4. Коммит:
   - Формат сообщения: `<TYPE>(<SCOPE>): <кратко что сделано>`
     - `TYPE` — согласованный префикс (например, `СВОР` или `feat`).
     - `SCOPE` — схема/область, например `ComparativeQuantityTakeoff`.
   - Пример: `СВОР(ComparativeQuantityTakeoff): обязательный fileCheckSum (CRC32)`.
5. Pull/Push:
   - `git pull`; при конфликтах — остановиться и сообщить.
   - `git push` после успешного pull и отсутствия конфликтов.

Если на любом шаге возникает неопределённость (новее/старее, конфликт, лишние изменения) — остановиться и запросить решение.
