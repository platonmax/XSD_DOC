# Задача 1186 — ExplanatoryNoteEstimate (ПЗ), визуалка, адрес объекта

## Контекст
В HTML-визуализации ПЗ в пункте 1.5 («Почтовый (строительный) адрес…») при заполненном только регионе отображается лишняя запятая после региона (например: `Регион 48,`).

## Как понял задачу (архитектор)
- В XSL есть единая точка форматирования адреса (`formatAddress`), которая “склеивает” части адреса через `, `.
- Сейчас разделитель `, ` добавляется без учёта того, есть ли дальше ещё заполненные части адреса. Если следующей части нет, разделитель становится «хвостом» (лишняя запятая).
- Требуемое поведение: выводить запятую **только** когда действительно есть следующая часть адреса; если заполнен только регион (или любой другой одиночный компонент), строка должна заканчиваться на компоненте без хвоста `,`.
